— Método de Resolução —
Branch and Bound
1 Relaxação Linear: resolve o problema ignorando a regra de inteiros temporariamente. Obtém solução fracionária — limite superior do lucro.
2 Identifica fração: x₃ = 0.97 — "Não dá hackear 0.97 vezes!" O resultado não é inteiro, então não pode ser usado.
3 Branch: cria dois mundos paralelos — x₃ ≤ 0 (não faz o hack) e x₃ ≥ 1 (faz pelo menos 1 hack). Cada ramo é um subproblema independente.
4 Poda: descarta ramos inviáveis (violam restrições) ou com lucro menor que a melhor solução inteira já encontrada.
— Análise Matemática —
De Onde Vem o 0.97?
A relaxação linear com a conta exata:
①
Fixamos os obrigatórios: x₁ = 2 e x₄ = 1
Suborno usado = 8(2) + 50(1) = $16k + $50k = $66k
②
Calculamos o suborno disponível restante
Disponível = $100k − $66k = $34k
③
Custo de 1 hack no sistema bancário
x₃ custa $35k — mas só temos $34k!
④
O Simplex (relaxação linear) calcula a fração máxima possível
x₃ = $34k ÷ $35k = 0.97  ← decimal, não é inteiro!
Por isso o Simplex sozinho não resolve — precisa do Branch and Bound para garantir valores inteiros.
— Resolução —
Resolução Passo a Passo
1 x₁ = 2 roubos de veículos (obrigatório) → usa 30h e $16k  →  sobram 90h e $84k
2 x₄ = 1 sequestro (o mais lucrativo) → usa 50h e $50k  →  sobram 40h e $34k
3 Tentar x₃ = 1 hack no sistema bancário → precisaria de $35k mas só temos $34k disponíveis.  → INVIÁVEL — ramo podado! ✗
4 x₅ = 2 corridas ilegais → usa 40h e $20k  →  recursos esgotados.  → VÁLIDO ✓
Z = 25(2) + 130(1) + 40(2)  =  50 + 130 + 80
= $260.000

O Que Aprendemos com o Golpe?
Variáveis de Decisão
Representam o que podemos controlar — quantas vezes cada crime é executado. São a base de toda modelagem em Programação Linear Inteira.
Função Objetivo
Define o que queremos maximizar ou minimizar — o lucro total do bando. Toda decisão aponta para esse norte.
Restrições
Os limites que não podem ser violados — tempo, suborno, reputação. A solução ótima vive na fronteira viável dessas restrições.
Branch and Bound
Garante a solução ótima inteira explorando e podando o espaço de busca. Mais eficiente que testar todas as combinações possíveis.
