O Problema
Uma nutricionista precisa elaborar um plano alimentar diário para um paciente adulto respeitando as recomendações mínimas da OMS para nutrientes e um orçamento máximo de R$20,00 por dia. Os alimentos disponíveis são: arroz, feijão, frango, ovo e brócolis.
Objetivo
Minimizar o custo total do plano alimentar diário, determinando a quantidade ótima — em porções de 100g — de cada alimento.
Tipo de Problema
Programação Linear de minimização com variáveis contínuas.

Resolução — Questões 1 e 2
Questão 1
Qual a solução ótima? Quais alimentos e em que quantidades?
Solução ótima pelo Simplex:
  • x₁ = 8,93 porções de arroz (893g)
  • x₂ = 1,47 porções de feijão (147g)
  • x₃ = 0  (frango não entra)
  • x₄ = 0  (ovo não entra)
  • x₅ = 2,15 porções de brócolis (215g)
Custo mínimo: Z = R$ 5,18 / dia
Questão 2
Se o preço do frango aumentar 20%, o plano muda?
Análise de sensibilidade:
  • Novo custo do frango: R$1,20 × 1,20 = R$ 1,44
  • x₃ = 0 já era a solução original
O aumento de 20% não altera a solução nem o custo. O plano permanece em R$5,18/dia com arroz, feijão e brócolis como base.
Questão 3 + Conclusão
Questão 3
Qual o impacto de eliminar o frango (x₃ = 0)?
Como x₃ = 0 já era a solução ótima original, eliminar o frango não altera o plano. A solução permanece viável. O nutriente mais comprometido é o cálcio — que depende principalmente do brócolis e feijão para atingir os 1000mg mínimos.
Conclusão
  • O Simplex encontrou um plano alimentar completo por apenas R$5,18/dia
  • O frango não entrou na solução por ter custo elevado em relação ao benefício nutricional
  • O cálcio foi o nutriente mais difícil de atingir — restrição mais crítica do modelo

— Método de Resolução —
Branch and Bound
1Relaxação Linear: resolve o problema ignorando a regra de inteiros temporariamente. Obtém solução fracionária — limite superior do lucro.
2Identifica fração: x₃ = 0.97 — "Não dá hackear 0.97 vezes!" O resultado não é inteiro, então não pode ser usado.
3Branch: cria dois mundos paralelos — x₃ ≤ 0 (não faz o hack) e x₃ ≥ 1 (faz pelo menos 1 hack). Cada ramo é um subproblema independente.
4Poda: descarta ramos inviáveis (violam restrições) ou com lucro menor que a melhor solução inteira já encontrada.
— Análise Matemática —
De Onde Vem o 0.97?
A relaxação linear com a conta exata:
①
Fixamos os obrigatórios: x₁ = 2 e x₄ = 1
Suborno usado = 8(2) + 50(1) = $16k + $50k = $66k
②
Calculamos o suborno disponível restante
Disponível = $100k − $66k = $34k
③
Custo de 1 hack no sistema bancário
x₃ custa $35k — mas só temos $34k!
④
O Simplex (relaxação linear) calcula a fração máxima possível
x₃ = $34k ÷ $35k = 0.97  ← decimal, não é inteiro!
Por isso o Simplex sozinho não resolve — precisa do Branch and Bound para garantir valores inteiros.
— Resolução —
Resolução Passo a Passo
1x₁ = 2 roubos de veículos (obrigatório) → usa 30h e $16k  →  sobram 90h e $84k
2x₄ = 1 sequestro (o mais lucrativo) → usa 50h e $50k  →  sobram 40h e $34k
3Tentar x₃ = 1 hack no sistema bancário → precisaria de $35k mas só temos $34k disponíveis.  → INVIÁVEL — ramo podado! ✗
4x₅ = 2 corridas ilegais → usa 40h e $20k  →  recursos esgotados.  → VÁLIDO ✓
Z = 25(2) + 130(1) + 40(2)  =  50 + 130 + 80
= $260.000
